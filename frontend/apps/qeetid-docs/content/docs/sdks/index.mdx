Qeet ID SDKs are first-class: same versioning, same release schedule, same support tier.
All clients are generated from the OpenAPI spec, so the surface area is identical across
languages.

## What you get in every SDK

- **Type-safe API client** generated from OpenAPI
- **Session validation** with edge caching
- **Permission checks** (`can()`, `canAll()`)
- **Webhook signature verification**
- **OpenTelemetry traces and metrics**
- **Idiomatic error types** (no `try { JSON.parse } catch`)

## Versioning

SDKs follow semver. Major versions track the Qeet ID platform major; minor and patch
versions match the OpenAPI spec they were built from.

```
SDK 1.4.x  ↔  API v1.4.x
SDK 2.x.x  ↔  API v2.x.x
```

Breaking changes are announced 90 days in advance via the [changelog](/docs/changelog)
and the `Sunset` header on deprecated endpoints.

## Reporting issues

Each SDK has its own GitHub repo under [github.com/qeetid](https://github.com/qeetid). File
issues there. For platform-level bugs, [support@qeetid.com](mailto:support@qeetid.com).
